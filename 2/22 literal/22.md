在 TypeScript 中，**Literal（字面量型別）**指的是「值本身就是型別」。
它將某個具體值（字串、數字、布林）收斂成只允許那個值。

---

## 1. 字面量型別的核心概念

一般型別：`string` 表示任何字串。
字面量型別：`"ok"` 只能是字串 `"ok"`。

---

## 2. 基本示例

```ts
let status: "success";
status = "success"; // OK
status = "error"; // NG
```

`"success"` 本身成為型別。

---

## 3. 字面量組合（最常用）

搭配 union（聯合型別）用來定義固定選項集合：

```ts
type LoadingStatus = "loading" | "success" | "error";
```

`LoadingStatus` 只能是這三者之一。

---

## 4. 字面量可用於數字、布林

```ts
let port: 80 | 443;
let flag: true;
```

---

## 5. 典型使用場景

- 限制參數只接受特定字串（如 API mode）
- 定義動作、事件、狀態常數
- 取代 enum 作為簡化選項集合
- Redux action type、按鈕樣式類型等

---

## 6. 舉例：函式參數限制

```ts
function setDirection(dir: "left" | "right" | "up" | "down") {
  return dir;
}

setDirection("left"); // OK
setDirection("abc"); // NG
```

---

## 7. Literal 的意義

把「值」收斂成「型別」。
避免任何字串亂入，提升型別安全性。

const vs let
在「字面量型別」語境下，`const` 與 `let` 的差異集中在「是否保留字面量型別」。

---

## 1. `let`：型別會被寬化（Widening）

```ts
let x = "yes";
```

TS 會把 `"yes"` 變成一般的 `string`，不會保留字面量型別。

結果：
`x` 的型別是 `string`，可重新指定任意字串。

---

## 2. `const`：型別不會被寬化，維持字面量型別

```ts
const y = "yes";
```

TS 直接把型別定義為 `"yes"`。

結果：
`y` 型別是字面量 `"yes"`，不可被賦值成其他字串。

---

## 3. 對 union 的影響

使用 `const` 才能正確保留字面量，方便組成穩定的 union。

```ts
const dir = "left";
// dir 的型別是 "left"

let mode = "left";
// mode 的型別是 string（寬化）
```

---

## 4. 若使用 let 但想保留字面量型別

需手動標註：

```ts
let status: "on" | "off" = "on";
```

---

## 5. 核心結論

`let`：值會被推斷為一般型別，字面量消失。
`const`：值被固定，字面量型別得以保留。
