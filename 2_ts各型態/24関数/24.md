**関数（函式）**、**パラメータ（參數）**、**戻り値（回傳值）**在程式中分別扮演不同角色。以下以中文精確定義並擴展其概念層級。

---

## 1. 関数（函式）

函式是「可重複呼叫的邏輯單元」。
將一段特定處理流程封裝起來，給予名稱，使其在程式中可以多次使用。

核心特徵：

- 接受輸入（參數）
- 執行內部邏輯
- 返回結果（回傳值，可有可無）

示例：

```ts
function add(a: number, b: number) {
  return a + b;
}
```

`add` 是一個計算加總的函式。

---

## 2. パラメータ（參數）

參數是「呼叫函式時輸入的資料」。
函式透過參數取得外部資訊，根據不同輸入執行不同處理。

參數特徵：

- 定義於函式小括號內
- 在函式內被視為變數使用
- 呼叫時必須提供對應的值（除非有預設值）

示例：

```ts
function greet(name: string) {
  console.log("Hello, " + name);
}
```

`name` 是參數，決定輸出的內容。

更複合的參數案例：

```ts
function fetchUser(id: number, verbose: boolean = false) {
  // id：必要參數
  // verbose：具預設值，可省略
}
```

---

## 3. 戻り値（回傳值）

回傳值是「函式執行後輸出的資料」。
函式透過 `return` 將結果傳回呼叫者，使外部可以利用此結果進行後續處理。

特性：

- 每個函式可有 0~1 次回傳值
- 若省略 `return`，回傳值為 `undefined`
- 回傳值的型別由函式簽名明確定義

示例：

```ts
function multiply(a: number, b: number): number {
  return a * b;
}
```

這個函式的回傳值型別為 `number`。

無回傳值的情況：

```ts
function logMessage(msg: string): void {
  console.log(msg);
}
```

`void` 表示沒有實際回傳任何資料。

多層次返回結構：

```ts
function getProfile(): { id: number; name: string } {
  return { id: 1, name: "Alice" };
}
```

---

## 4. 三者關係（總結）

- 函式：邏輯單元
- 參數：函式的輸入
- 回傳值：函式的輸出

概念關係可表述為：

```
輸入（參數） → 處理（函式） → 輸出（回傳值）
```
